# MNGA homepage - Claude 運用ルール

このリポジトリで作業するときのデフォルト動作。

## 自動コミット・プッシュ・デプロイ

ユーザーから依頼されたタスク（機能追加・修正・ドキュメント更新など）が
**正常に完了した時点で、ユーザーに確認せず以下を自動実行する**:

1. 変更したファイルを `git add` で個別指定してステージング
   （`git add .` や `-A` は使わない。意図せぬファイルの混入を防ぐため）
2. 変更内容を端的に表す Conventional Commits 形式のメッセージで `git commit`
3. `git push origin main` でリモートに反映
4. GitHub Actions が Vercel へ自動デプロイするので、追加作業は不要

この運用を明示的に無効化したいとき、ユーザーは
「コミットしないで」「まだプッシュしないで」と伝える。

## 自動化しない操作（必ず確認）

以下は自動実行しない。必ずユーザーに確認する。

- `git push --force` / `-f` などの破壊的プッシュ
- `git reset --hard` / `git rebase -i`
- 本番 DB（Supabase）のスキーマ変更を伴う SQL 実行
- 依存関係の大幅な変更（`npm install` で新規パッケージ追加など）
- `CLAUDE.md` / `.github/workflows/*` / `vercel.json` / `.gitignore` の変更
  （設定ファイルは挙動全体に影響するので確認を挟む）

## コミットメッセージ規約

Conventional Commits を使う:
- `feat:` 新機能
- `fix:` バグ修正
- `chore:` 雑務・リファクタ
- `docs:` ドキュメントのみ
- `security:` セキュリティ関連
- `style:` 見た目・CSS のみ

件名は英語 or 日本語どちらでも可。本文があるときは空行を挟む。

## タスクを「完了」とみなす条件

- コードを書いた場合: 型エラー・Lint エラーが出ていない
- UI 変更の場合: ローカルで確認できる変更であること（デプロイ後の確認はユーザー側）
- 複数ファイルをまたぐ作業: 全ファイル更新し終えたタイミング

途中でのコミットはしない。

## デプロイ先

- 本番: Vercel（`main` ブランチ push で自動デプロイ）
- GitHub Actions の `VERCEL_TOKEN` は設定済み
