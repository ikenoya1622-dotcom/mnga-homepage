## 5. インフラ・リリース準備

### 5-1. ドメイン取得
- [ ] ドメイン名の決定（例: `mnga.jp` / `mnga.or.jp` / `mnga-japan.com` 等）
  - `.jp` / `.or.jp`（法人向け）/ `.com` の選択肢を比較
  - `or.jp` は一般社団法人・NPO等に限定、登記証明が必要
- [ ] ドメインレジストラの選定・アカウント作成
  - 国内推奨: お名前.com / ムームードメイン / さくらのドメイン
  - 海外推奨: Cloudflare Registrar（コスト最安）/ Namecheap
- [ ] ドメインの購入・登録
- [ ] Whois 情報（登録者名・住所）の設定（法人名義にするか確認）
- [ ] ドメイン自動更新の設定

### 5-2. ホスティング（サーバー）環境の選定・契約
> このサイトは React の静的ビルド成果物のため、従来のレンタルサーバーは不要。
> 静的ホスティングサービスの利用を推奨。

**推奨: Vercel（無料プランで十分）**
- [ ] Vercel アカウント作成（GitHub アカウントと連携）
- [ ] GitHub リポジトリへのプッシュ確認
- [ ] Vercel にプロジェクトをインポート・初回デプロイ確認
- [ ] 環境変数（`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`）を Vercel ダッシュボードに設定
- [ ] 本番ビルドの動作確認（`npm run build` エラーがないか）

**代替案**
- Cloudflare Pages（無料・高速 CDN）
- Netlify（無料・日本語ドキュメントあり）

### 5-3. DNS 設定（ドメインとホスティングの紐付け）
- [ ] ホスティングサービス（Vercel 等）が指定するネームサーバーまたは CNAME/A レコードを確認
- [ ] ドメインレジストラの DNS 管理画面でレコードを設定
  - Vercel の場合: `A` レコード × 2 または `CNAME` レコードを追加
- [ ] DNS 反映待ち（最大 24〜72 時間）の確認
- [ ] ドメインでサイトが表示されることを確認

### 5-4. HTTPS（SSL/TLS）設定
- [ ] Vercel / Cloudflare Pages 等は自動発行のため、証明書の自動適用を確認
- [ ] `http://` アクセスを `https://` に自動リダイレクトされることを確認
- [ ] 証明書の有効期限・自動更新設定の確認

### 5-5. Supabase 本番設定
- [ ] Supabase プロジェクトの「本番用」設定を確認
  - 現在の無料プランの制限（500MB DB / 1GB Storage / 50,000 MAU）を確認
  - 必要に応じて Pro プランへのアップグレード検討（$25/月〜）
- [ ] Supabase の「Site URL」を本番ドメインに変更（Auth 設定）
- [ ] Supabase の `Allowed Redirect URLs` に本番ドメインを追加
- [ ] Row Level Security（RLS）ポリシーの確認
  - `reports` テーブル: 一般ユーザーは読み取りのみ、管理者のみ書き込み可
  - Storage バケット: 公開読み取り、アップロードは認証済みのみ
- [ ] API キー（anon key / service_role key）の管理・漏洩防止確認

### 5-6. メール送信環境（問い合わせフォーム用）
- [ ] 送信元メールアドレスの決定（例: `info@mnga.jp`）
- [ ] メール送信手段の選定
  - **Resend**（無料 3,000通/月、Supabase との相性良好）
  - **SendGrid**（無料 100通/日）
  - **Formspree**（フォーム専用、月100件まで無料）
- [ ] 選定したサービスのアカウント作成・API キー取得
- [ ] SPF / DKIM レコードの DNS 設定（迷惑メール対策）
- [ ] テスト送信確認

### 5-7. 最終デプロイ・公開
- [ ] 本番ビルド（`npm run build`）でエラーがないことを確認
- [ ] ステージング環境（Vercel Preview URL 等）での最終動作確認
- [ ] 本番ドメインへの切り替え・公開
- [ ] Google Search Console へのサイト登録・所有権確認
- [ ] sitemap.xml の Search Console への送信
